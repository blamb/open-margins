The design problem

OER removed
the cost barrier.
It didn't remove
the reading problem.

Open educational resources solved the access problem. A free, openly licensed textbook left sitting in a browser tab, unread and unexamined, solved nothing. The engagement problem is different from the access problem — and tools built for one don't automatically serve the other.

Open Margins is a toolkit that tries to do something specific: embed active, inquiry-based engagement directly into the act of reading OER. Five instruments. Each one built around a different theory of what it means to do something with a text rather than just move your eyes across it. Sylva builds Bloom's-tagged lesson arcs in minutes. Nova produces shareable concept maps and inquiry activities students open in any browser. The Activity Builder outputs LMS-ready H5P activities. Rhizo helps students question what a text is taking for granted.

Works with any openly licensed OER — TRU Open Press titles, external URLs, PDFs, Word documents, or pasted text. Multiple sources at once, for cross-text synthesis. Designed for online and asynchronous contexts. Student activity URLs work without accounts, API keys, or institutional login infrastructure.

note
Built in alignment with the TRU Learning Design Framework — caring, open, connected, active. Not as branding. As an actual design constraint. The framework was developed by TRU Open Learning's instructional design community and means something to the people who built these tools.

01 · Companion

Companion

Reading alongside, not instead of.

Companion lives in the sidebar while someone reads an OER. Highlight a passage and it can illuminate the text in plain language — using analogies drawn from the chapter itself rather than reaching for outside explanations. It can surface what a sentence is resting on, or generate a question worth sitting with. The key word is alongside: it scaffolds the reading, it doesn't replace it.

Every response — AI or personal annotation — saves in the session. Export as Markdown: portable, open format, not locked in any platform. A reading record that belongs to the reader.

  • Illuminate — plain-language explanation with analogies drawn from the text itself
  • Interrogate — surfaces what a passage is assuming rather than arguing
  • Annotate — student adds their own marginal notes alongside AI responses
  • Export — everything saved as open Markdown, not locked in any vendor's format
  • Works on any OER URL, PDF, Word doc, or pasted text

02 · Activity Builder

Activity Builder

Formalize what good instructors already do.

Activity Builder generates H5P learning activities from any OER source material. H5P is worth a mention here: it is genuinely open source, widely supported across LMS platforms, and not going anywhere. The tool produces eight activity types. An optional learning objective field shapes what the AI prioritizes — that objective carries into the H5P metadata.

  • Quiz, true/false, flashcards, fill-in-the-blank, matching, short answer, ordering, scenario
  • Optional learning objective field — focuses AI output, carries into H5P metadata
  • H5P export — open source, LMS-compatible, no proprietary lock-in
  • Multiple source documents for cross-text synthesis activities
  • Outputs are editable before export — the AI generates a draft, you decide what ships
note
The learning objective field isn't decoration. Feed it a specific, well-formed objective and the activity that comes out reflects your pedagogical intentions rather than the model's generic defaults. Bloom's verb and all.

03 · Nova

Nova

Knowledge has a shape. Let's see it.

Nova generates eight types of learning activities from an OER source. The most visually striking is the Concept Map — an interactive node diagram showing how the ideas in a chapter connect, diverge, and return. Share the URL and students open an interactive version in any browser, no installation, no API key. Each activity includes a live chat interface for following up on whatever the activity surfaces.

  • Concept Map — interactive diagram of ideas and relationships in the text
  • Socratic Tutor — a question sequence that earns answers rather than providing them
  • Evidence Tribunal — students evaluate claims against the text's own evidence
  • Analogy Forge — builds bridges between the text and what students already know
  • Concept Timeline — places ideas in historical and developmental sequence
  • Assumption Excavator — makes explicit what the text takes for granted
  • Transfer Challenge — applies concepts outside the original context
  • Question Generator — produces researchable questions the text doesn't fully answer

04 · Rhizo

Rhizo

Learning that grows sideways.

The name is a reference. Deleuze and Guattari's rhizome — the root that spreads laterally, that has no fixed centre, no single origin, no correct entry point. Dave Cormier's rhizomatic learning, which takes seriously what education might look like if knowledge were genuinely unmappable. The idea that a text, like a forest floor, contains more connections than any diagram can hold — and that the edges are often where the most interesting things are happening.

Rhizo generates eight activity types that take the edges of a text seriously: its assumptions, its silences, the perspectives it crowds out, the questions it forecloses. Each activity includes a live chat thread for following up. Shareable via URL — students need nothing installed.

  • Weed the Garden — identify and challenge unexamined assumptions in the text
  • Uncertainty Map — chart what the text doesn't know it doesn't know
  • Wicked Council — surface perspectives the text marginalizes or excludes
  • Nomad's Entry — approach the text as an outsider; what does a first reading miss?
  • Community Contract — examine the implicit social agreements the text enforces
  • Trust Audit — ask which voices the text treats as authoritative and why
  • Abundance Check — find what the text opens without following through
  • Breadcrumb Trail — trace the lineage of ideas; where did this argument come from?

05 · Sylva

Sylva

Bloom's as scaffold, not verdict.

Sylva generates a lesson arc from any OER source material — introduction, core activities, discussion prompts, assessment ideas — in three delivery modes (in-class, online synchronous, asynchronous). Before it builds the arc, it generates 3–5 learning objectives tagged to Bloom's Taxonomy. The word editable matters here.

Bloom's is a framework, not a formula. Revise the objectives before the arc is built and the output follows your pedagogical intentions rather than the model's first guess. Includes constructively-aligned assessment suggestions, meaning the assessments connect back to the objectives rather than floating free of them.

  • 3–5 Bloom's Taxonomy-tagged objectives — editable before the arc is generated
  • Three delivery modes: in-class, online synchronous, asynchronous
  • Constructively-aligned assessment suggestions (Biggs, 1996)
  • Works across multiple OER sources for unit-level planning
  • Full lesson arc output — introduction through assessment, in sequence

The values this runs on

Caring. Open.
Connected. Active.

The TRU Learning Design Framework names four values that should run through every design decision. These tools were built to those values. Whether they succeed is, as always, your assessment to make.

◎
Caring
Learners are not users. They are people who get stuck, who bring their whole contexts into a course, who deserve tools that meet them where they are rather than process them through a workflow. Companion is the most literal expression of this value — it reads alongside the student rather than for them.
◻
Open
Open source where possible. Open licensing always. CC BY on the tools themselves. Built on open educational resources. H5P outputs rather than proprietary formats. No lock-in to any vendor who will be acquired within 18 months. The open web was always an argument about power, not just technology.
⬡
Connected
Learning happens in relation — to texts, to other learners, to the instructor, to the world outside the course shell. Rhizo and Nova surface connections a first reading misses. Shareable activity URLs let instructors put students in the same conceptual space without requiring them to be in the same room.
✦
Active
Engagement is not the same as activity. Reading is active. Questioning is active. Mapping a concept is active. The tools try to build those acts into the structure of the course — not bolt them on after the passive consumption is done. The difference matters.

The part we want you to know about

Something to say,
despite everything.

Here is what most software marketing sites don't say about their AI integration. These tools use the Anthropic Claude API. That means a commercial product from a San Francisco company backed by some of the largest technology investors in the world. The environmental costs of training and running large language models are real and inadequately disclosed by the parties most invested in minimizing them. The labor conditions involved in generating and rating AI training data are documented and worth knowing about.

We built these tools because we think the pedagogical case for them is genuine — and because the alternatives in institutional edtech are, on balance, worse on privacy, worse on data practices, and carry no commitment to open education whatsoever. That is a judgment call. You are entitled to weigh it differently.

The decision to use AI in education is not simple. Treating it as simple in either direction — transformative revolution, or categorically inadmissible — skips over the actual work. We tried to do the actual work.

What we actually did about it
  • Your API key is stored locally — it never passes through our servers, only from your machine directly to Anthropic
  • No student data is transmitted without your explicit action — students using shared activity URLs make no API calls at all
  • Activity URLs work without further AI calls once generated — the output lives at the URL, not in an ongoing session
  • The tools are MIT-licensed open source — not a proprietary platform. You can read the code, fork it, host it yourself, change it
  • No usage analytics, no behavioural tracking — we are not in the business of knowing what your students do with these tools
  • Everything runs on your infrastructure — your computer, your department's server, your institution's network. Not ours.
Getting started · deployment

Run on your own computer or a shared departmental server

Open Margins is a small web application that runs on your own computer — or on a server your department or institution controls. No data is stored externally. The only outbound connection is the text you send to Claude for analysis, which travels directly and privately to Anthropic. No third-party accounts, no vendor subscriptions, no student data collected.

To run the tools, you need two things. First, Node.js — a free program that acts as a lightweight local web server on your computer, letting the tools talk to Claude securely without exposing your API key. Second, an Anthropic API key — your personal access credential for Claude, billed on a pay-per-use basis at roughly $5–10 for a full semester of active use. Both take about ten minutes to set up.

For departmental or institutional deployment, Open Margins can run on any server — your IT team hosts it once, and instructors access everything through a shared URL. Students never need to install anything.

Node.js v18+ — a free JavaScript runtime; acts as a local web server on your computer (nodejs.org)
An Anthropic API key — pay-per-use access to Claude, ~$5–10/semester (console.anthropic.com)
The Open Margins project files — downloaded from GitHub or provided by TRU Open Press
A modern browser — Chrome, Firefox, Edge, or Safari
Optional: a departmental or institutional server for shared multi-user access
1
Get your Anthropic API key
Create a free account at console.anthropic.com and generate an API key. Treat it like a password. You're only billed for what you use — typically a few cents per session, totalling $5–10 over a semester of regular use.
2
Install Node.js on your computer
Download and install Node.js v18 or later from nodejs.org. Node.js is a free program that acts as a tiny web server on your machine, securely routing requests between the tools and Claude without exposing your API key to students or the browser.
3
Tell your computer your API key, then install the tool libraries
Open Terminal (Mac) or Command Prompt (Windows). Paste in your key, then run npm install in the project folder. Downloads the support libraries the tools depend on — takes about 30 seconds and only needs to be done once.
export ANTHROPIC_API_KEY=sk-ant-…
4
Start the background servers (two terminal windows)
Run node server.js in the project folder for Activity Builder, Nova, Rhizo, and Sylva. Open a second terminal window, navigate into the companion/ subfolder, and run the same command for Companion. Both servers stay running silently while you use the tools.
5
Open the tool launcher in your browser — you're ready
Open index.html in your browser (or go to localhost:3001) to see the tool launcher. Activities generate shareable URLs — paste them into your LMS for students to open directly. Close the terminal windows when you're done for the day.
